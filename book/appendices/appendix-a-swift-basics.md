# 附錄 A Swift 最小基礎

## 這份附錄是寫給誰的

這份附錄不是要把 Swift 從零完整教完，而是要替那些「已經能看懂一點程式，但還不夠穩」的讀者補票上車。

如果你在讀正文時，常常會卡在這些地方：

- `let` 和 `var` 到底差在哪裡
- `struct` 為什麼一直出現
- `enum`、函式、陣列和迴圈看得懂一半
- 閉包和 optional 一出現，腦中就開始斷線

那你可以先把這份附錄讀完，再回到正文。

## 先有一個最小觀念

學 SwiftUI 前，不需要先把整個 Swift 都學到很深；但你至少要能分辨：

- 值是什麼
- 型別是什麼
- 一段資料要怎麼被組起來
- 一個行為要怎麼被包成函式

正文會一直依賴這些能力，所以這份附錄只補最常反覆出現、而且最容易卡住的地方。

## 1. `let` 與 `var`

先記一個很夠用的版本：

- `let` 表示建立後不打算再改的值
- `var` 表示後面可能還會改的值

```swift
let title = "今天"
var completedCount = 0
```

當你看到 SwiftUI 畫面裡有些值用 `let`，有些值用 `@State var`，本質上也是同一個問題：這個值後面還會不會改？

## 2. `struct`

SwiftUI 裡 `struct` 會一直出現，因為很多資料和畫面都用它來描述。

```swift
struct Habit {
    let name: String
    var isCompletedToday: Bool
}
```

你可以先把它理解成：

`struct` 是把一組彼此有關的資料包在一起的方法。

在正文裡，你會看到：

- 用 `struct` 表示一筆習慣資料
- 用 `struct` 表示一個 SwiftUI `View`

它們長得像，是因為本質上都在做同一件事：描述一個清楚的單位。

## 3. `enum`

`enum` 很適合用來表示「有限種狀態」。

```swift
enum LoadState {
    case idle
    case loading
    case loaded
    case failed
}
```

這在 SwiftUI 很常用，因為畫面本來就常常需要根據不同狀態顯示不同結果。

如果你記不住所有細節，也先不用急，只要先抓住一個核心：

`enum` 很適合拿來描述某件事目前處在哪一種可能情況。

## 4. 函式

函式就是把一段會重複做的事情包起來。

```swift
func toggleCompleted() {
    isCompletedToday.toggle()
}
```

在正文裡，函式常常會用來：

- 切換狀態
- 提交表單
- 載入資料
- 寫回本地

你可以先用最樸素的方式理解：

`函式 = 替一段行為取名字。`

## 5. 陣列與迴圈

陣列就是把多筆同型別資料排成一列。

```swift
let habits = ["晨間伸展", "閱讀 20 分鐘", "晚間散步"]
```

當你想把它們一筆一筆拿出來處理，就會遇到迴圈。

```swift
for habit in habits {
    print(habit)
}
```

SwiftUI 裡會更常看到 `ForEach`，但它的閱讀感跟這裡很接近：都是在處理「一組資料中的每一筆」。

## 6. Optional

optional 的意思很簡單：

`這個值有可能存在，也有可能不存在。`

```swift
var note: String?
```

這表示 `note` 可能有文字，也可能是空的。

正文中你最常遇到 optional 的地方通常是：

- 某筆資料可能還沒選中
- 某個欄位可以留白
- 某段非同步資料還沒回來

一開始先建立直覺就夠：

- 有值：裡面真的裝了東西
- 沒值：現在是 `nil`

## 7. Closure

closure 常常是初學者第一個覺得「突然變抽象」的地方，但在這本書裡，你可以先用更務實的方式理解：

`closure 就是一段可以被傳來傳去的行為。`

例如：

```swift
Button("標記完成") {
    isCompletedToday.toggle()
}
```

大括號裡那段，就是 closure。

在正文裡，closure 很常被用來：

- 按鈕被點時要做什麼
- 子視圖通知父視圖要做什麼
- 某段非同步流程完成後要做什麼

如果你還不熟，不必一開始就追完整個理論，先把它看成「被包起來的一段行為」就很好用。

## 8. 你讀正文前至少要熟到哪裡

如果你想順暢讀完這本書，建議你至少要做到：

- 看得懂 `struct`
- 看得懂函式和基本條件判斷
- 知道陣列裡有多筆資料
- 知道 `let` 和 `var` 的差別
- 遇到 optional 和 closure 時，不會完全看不下去

如果這幾件事已經有了，正文就可以跟上。

## 9. 讀這本書時怎麼補這份附錄最有效

最不建議的讀法，是把附錄 A 當成一門獨立課程硬背完。

更有效的方式通常是：

1. 先直接進正文。
2. 卡住時回來補這份附錄。
3. 補完之後立刻回正文，看那個語法現在是不是突然清楚了。

這樣你學到的不是孤立知識，而是會立刻用回本書主線上的知識。
