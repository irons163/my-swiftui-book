# 前言

如果你正在學 SwiftUI，你很可能已經走過一段既熟悉又挫折的路。

你也許已經看過不少教學，知道 `Text`、`Button`、`List`、`NavigationStack` 這些元件怎麼用；你也許能跟著影片把畫面做出來，甚至做出一個看起來像樣的小作品。但只要需求一變、畫面一多、資料開始流動，原本還算順手的程式就很容易突然變得很重。這時候最讓人不安的，通常不是語法不會，而是你會開始懷疑：

- 我現在這樣切畫面，到底穩不穩？
- 這份資料應該放哪裡，誰來改？
- 為什麼畫面能跑，卻還是覺得整個專案沒有長在一起？

我想寫這本書，就是因為我越來越確定，很多人真正缺的不是更多 API 清單，而是一條能把零散知識串成完整開發思路的路。

SwiftUI 最難的地方，往往不是元件名稱，而是思考方式。它要求我們重新理解幾件事情：畫面和狀態的關係、資料應該如何流動、版面應該怎麼組合、專案變大後又該如何維持清楚。只要這幾條線沒有想通，學再多範例也很容易停在「會照著做」，卻還沒有走到「知道自己在做什麼」。

因此，這本書不打算把 SwiftUI 寫成一份語法目錄，也不打算一開始就把讀者推進過重的架構世界。老實說，我不太相信那種一開始就把所有抽象都攤在讀者面前的教法。它看起來很完整，實際上卻常常讓人更快失去方向。這本書想做的事情比較單純，也比較實際：

`帶你用一條清楚而可重複的路，從第一個畫面開始，一路走到一個真正像樣的 App。`

書裡會用一個主線專案貫穿全書，也就是「習慣養成 App」。我選這個題目，不是因為它特別花俏，而是因為它足夠真實，又剛好能把 SwiftUI 最重要的幾條能力自然帶出來：

- 版面配置與資訊階層
- 狀態與資料流
- 清單、表單與導航
- 動畫與互動回饋
- 非同步資料與持久化
- 架構整理、測試與產品整合

換句話說，這本書不只會帶你看範例，而是會陪你看著一個 App 一章一章長起來。你會看見它怎麼從一張可互動的卡片，慢慢變成一個有首頁、有列表、有統計、有資料生命週期，也有版本邊界感的產品原型。我希望你讀到後面時，感受到的不是「我好像又多學了幾個技巧」，而是「原來一個 App 真的是這樣一步一步站穩的」。

我也希望這本書能一直回答三個問題：

- 這個觀念是為了解決什麼問題？
- 它在真實 App 裡會用在哪裡？
- 初學者最容易在哪裡誤會或踩坑？

如果你是完全沒有程式基礎的讀者，這本書可能不會是最輕的起點；但如果你已經有基本 Swift 語法，正在找一條能真正把 SwiftUI 學成「做產品能力」的路，那我很希望這本書能陪你走完這段關鍵轉換。

對我來說，一本技術書最有價值的地方，不是讓讀者看完後覺得作者很厲害，而是讓讀者真的開始相信：自己也做得到。真正有用的教學，不是把距離拉開，而是把路照亮。

如果這本書最後能幫你從「會跟著做」走到「能自己做」，從「畫面能跑」走到「專案站得住」，那它就完成了它的任務。
